# Traffic Groups admin UI — missing i18n namespace

> Status: `approved` · Phase: `P1` · Severity: cosmetic (renders raw key paths)

## Why

The traffic-groups admin feature (data model + service + REST API + Vue page + dialog component) was implemented in commit `28cfb08`. Backend works (verified live at `http://178.105.64.108:51821` after migration fix in commit TBD). However, the entire `admin.trafficGroups.*` i18n namespace is missing from `src/i18n/locales/en.json`, so every label, button, and dialog field renders as the literal key path (e.g. `admin.trafficGroups.createGroup`). The page is unusable to anyone who doesn't read the source.

Reference: `t(...)` calls in `src/app/pages/admin/traffic-groups.vue` and `src/app/components/Admin/TrafficGroupDialog.vue` reference 24 unique keys under `admin.trafficGroups.*`. None exist in `en.json`.

## Scope

### In

- Add the `trafficGroups` key under the existing `admin` object in `src/i18n/locales/en.json`.
- Provide English values for **exactly** the 24 keys listed below — no more, no fewer.
- Match existing en.json key style (concise sentence case, no trailing punctuation, consistent with neighbouring `admin.users.*` strings).

### Out

- Translations for non-English locales (separate PRD; English first to unblock the live deployment).
- Any change to Vue templates or backend code.
- Adding new copy not currently referenced in the templates.
- The standalone keys `common.loading`, `dialog.cancel`, `dialog.create`, `dialog.save`, `save`, `form.save` — these are shared/common keys outside this PRD's namespace and may already exist.
